# Getting Started

This guide helps you start using Actor-RTC Web quickly.

## Prerequisites

- Node.js 18+
- npm 9+
- A modern browser, preferably one of the latest two Chrome, Firefox, or Safari releases

## Usage Modes

### Client Mode

Client mode is the recommended starting point. A web application connects to remote actors and issues RPC-style calls or stream subscriptions.

Install the SDK:

```bash
npm install @actr/web
```

For React projects:

```bash
npm install @actr/web @actr/web-react
```

Recommended Vite setup:

```bash
npm install --save-dev vite-plugin-wasm vite-plugin-top-level-await
```

```ts
import { defineConfig } from 'vite';
import wasm from 'vite-plugin-wasm';
import topLevelAwait from 'vite-plugin-top-level-await';

export default defineConfig({
  plugins: [wasm(), topLevelAwait()],
  server: {
    headers: {
      'Cross-Origin-Opener-Policy': 'same-origin',
      'Cross-Origin-Embedder-Policy': 'require-corp',
    },
  },
});
```

Basic example:

```ts
import { createActor } from '@actr/web';

const actor = await createActor({
  signalingUrl: 'wss://signal.example.com',
  realm: 'demo',
});

const response = await actor.call('echo-service', 'sendEcho', {
  message: 'Hello, Actor-RTC!',
});
```

Stream example:

```ts
const unsubscribe = await actor.subscribe('metrics-service', (data) => {
  console.log(data);
});

await unsubscribe();
```

### Runtime Mode

Runtime mode runs a fuller Actor runtime in the browser. Use it only when you need browser-side actor execution, mailbox persistence, or more advanced transport orchestration.

At a high level:

- The DOM side manages browser APIs and UI-facing coordination.
- The Service Worker side hosts runtime state, transport management, and mailbox-driven processing.
- WASM modules provide the shared runtime logic.

## React Integration

Typical React integration uses `useActor` and service-specific hooks generated by codegen. Keep actor creation near the app boundary and pass stable references down through context or props.
